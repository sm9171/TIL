# Systemd Target

참고 블로그 : - 히포킹 (opens new window)

# Systemd란?

리눅스의 부트 프로세스이다. CentOS7 이전 버전의 runlevel과 비슷한 개념이다.

# Systemd Target 레벨

  1. poweroff.target: 시스템 종료
  2. rescue.target : single-user 모드(안전 모드)
  3. multi-user.target : 텍스트 모드의 multi-user 모드(텍스트 유저 모드)
  4. graphical.target : 그래픽 모드의 다중 사용자 모드
  5. reboot.target : 시스템 재부팅

# Systemd Target 정보 확인

$ systemctl get-default

# Systemd Target 변경

$ systemctl set-default multi-user.target

target을 multi-user.target으로 변경

# FTP ACTIVE MODE와 PASSIVE MODE의 차이점

참고 블로그 : - madplay님의 블로그 (opens new window)

# FTP란?

FTP는 File Transfer Protocol의 약자로 말그대로 파일을 전송하는 통신 규약이다.

  • 액티브 모드에서는 21번 포트로 접속한 후에 20번 포트로 데이터를 주고 받는다. 클라이언트가 서버에 접속을 하는 것이 아닌 서버가 클라이언트에 접속을 하는 것이다.

  • 패시브 모드에서는 앞선 Active 모드에서 사용했던 20번 포트를 사용하지 않고 1024번 이후의 임의의 포트를 데이터 채널 포트로 사용하게 된다.

# CI/CD

참고 영상 : - 얄팍한 코딩사전 - CI/CD가 뭔가요? (opens new window)

  • CI(지속적 통합) : 소스를 배포하기 전에 하는 테스트를 자동화 하는 것
  • CD(지속적 배포) : 사용자가 작업물을 실행 할 수 있는 상태까지 배포를 자동화 하는 것
  • 설치형 자동화 툴
  • 젠킨스 : 리눅스,맥,윈도우 상관없이 설치가능. 작업 콘솔을 띄어 설정 가능함. github같은 프로젝트 저장소와 연결하여 푸쉬하면 자동으로 젠킨스 폴더에 업로드 됨
  • 클라우드형 자동화 툴
  • travis CI : 젠킨스와 기능은 같으나 설지가 아닌 클라우드서버에서 작업
